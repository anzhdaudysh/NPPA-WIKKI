# НППА · Вики

Контентный проект на React + Vite. Стилистика выровнена под фирменный стиль
[openapc.ru](https://openapc.ru/) (носитель вики). Контент собран из материалов
рабочей группы ОАСУ ТП: openapc.ru/nppa, openapc.ru/collective-stand, описаний
распределённого стенда и карты компонентов.

## Запуск

```bash
npm install
npm run dev
```

Откроется на http://localhost:5173

## Сборка

```bash
npm run build
npm run preview
```

## Структура

```
src/
  data/
    content.js          — Обзор, Ключевые элементы, Принципы (с описаниями)
    international.js    — Международный опыт (NOA, OPAF, Китай)
    stand.js             — Архитектура стенда + Релизы
    components.js        — Карта компонентов (витрина решений)
    glossary.js           — Термины и аббревиатуры
    routeSections.js      — карта якорей по страницам (используется сайдбаром)

  components/
    Sidebar.jsx          — навигация: resizable, разворачивает главы активного раздела
    Footer.jsx
    SpecTag.jsx           — визуальный маркер ссылок на реестры/стандарты
    sections/             — переиспользуемые блоки контента:
      ElementsContent.jsx, PrinciplesContent.jsx, InternationalContent.jsx
      (используются и в Overview, и на отдельных страницах /elements, /principles, /international)

  pages/
    Overview.jsx          — главная: интро → принципы → элементы → межд. опыт → разделы
    Elements.jsx, Principles.jsx, International.jsx — те же блоки как отдельные страницы
    stand/StandArchitecture.jsx, stand/StandReleases.jsx
    Components.jsx        — интерактивная карта компонентов
    Glossary.jsx
    Contacts.jsx

  hooks/
    useAnchorScroll.js    — прокрутка к разделу страницы через ?to= (см. ниже)

  styles/global.css       — все дизайн-токены, стили, анимации
```

## Дизайн — стилистика openapc.ru

- Шрифты: **Archivo** (заголовки, широкий гротеск в духе фирменного стиля),
  **Inter** (текст), **JetBrains Mono** (spec-теги, коды стандартов)
- Палитра: тёмно-индиго `#1B1C3A` (фирменный navy), белый фон, светло-серые
  панели `#EFEFEC`, сигнальный оранжево-красный `#DD5B36` (акцент-кнопки, как CTA на openapc.ru)
- Компонент `<SpecTag>` — маркер ссылок на реестры/стандарты (РЭП, ЕРРП, ISA/IEC 62443 и т.п.)

## Навигация

- **Resizable-сайдбар**: тянуть за тонкую полоску на правом краю панели
  (двойной клик — сброс к ширине по умолчанию). Ширина сохраняется в localStorage.
- При переходе на раздел сайдбар автоматически показывает список его глав
  (см. `src/data/routeSections.js`) — клик по главе скроллит к ней на странице.
- Принципы / Ключевые элементы / Международный опыт встроены в `/` (Overview)
  одним полотном, но также остаются отдельными страницами-якорями в сайдбаре
  для прямых ссылок.

## Якоря внутри HashRouter

Поскольку роутинг уже использует `#` (`HashRouter`), обычные `<a href="#id">`
конфликтуют с роутингом. Внутристраничные переходы реализованы через
query-параметр: `/?to=principles`, `/stand/architecture?to=hub` и т.д.
Хук `useAnchorScroll` в `App.jsx` подхватывает `?to=` и плавно скроллит
к элементу с соответствующим `id`. При добавлении нового якоря — задайте
`id` на нужном элементе и пропишите его в `routeSections.js`.

## Плавность интерфейса (анимации)

Все анимации задокументированы прямо в конце `src/styles/global.css`
(раздел «ПЛАВНОСТЬ ИНТЕРФЕЙСА — как откатить, если не понравится»):
токены длительности в `:root`, точечные hover-transition, и 4 keyframe-анимации
входа (переход между страницами, раскрытие сайдбар-подсписка, блока архитектуры,
панели детали). Там же — быстрый общий рубильник на случай, если захочется
выключить всё разом.

## Роутинг

`HashRouter` (адреса вида `/#/stand/architecture`) — для простого деплоя на
статический хостинг без настройки SPA-фоллбэка. Если хостинг поддерживает
rewrite на `index.html`, можно заменить на `BrowserRouter` в `src/App.jsx`
(тогда якоря тоже можно будет упростить до обычных `#hash`).
